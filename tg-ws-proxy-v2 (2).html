Возможности
Всё что нужно, ничего лишнего

Приложение решает одну задачу и решает её хорошо — держит Telegram-прокси живым в фоне.

🔄
Foreground Service
Android Foreground Service гарантирует что прокси не будет убит системой. Постоянное уведомление — знак жизни сервиса.
🐍
Python прямо в APK
Через Chaquopy Python 3.11 встроен в сам APK. Ничего не нужно устанавливать дополнительно — ни Termux, ни скриптов.
🔁
Авторестарт
START_STICKY + AlarmManager перезапускают сервис если его убили. BootReceiver стартует прокси автоматически после перезагрузки телефона.
📊
Трафик в реальном времени
Статус-карточка показывает отправленные и полученные байты напрямую из ядра прокси. Обновление каждые 3 секунды.
🔒
Шифрование MTProxy
Библиотека cryptography собирается Chaquopy под Android автоматически — трафик зашифрован стандартом MTProxy.
⚡
Одно нажатие в TG
Кнопка «Открыть в TG» в уведомлении мгновенно передаёт настройки в Telegram. Никакого копирования ссылок вручную.

Под капотом

Три ключевых компонента, которые делают приложение надёжным.

Архитектура
Python-ядро внутри Android-оболочки
Ядро прокси — оригинальный Flowseal/tg-ws-proxy на Python, без единого изменения. Вокруг него Android-приложение на Java. Связующее звено — Chaquopy: он компилирует Python 3.11 и библиотеки прямо в APK для ARM64 и x86_64, так что на телефоне не нужна никакая Python-среда.

Мониторинг
Трафик из самого ядра прокси
Приложение читает stats.bytes_up и stats.bytes_down напрямую из Python-кода ядра — никаких костылей и перехватчиков. Данные передаются через Chaquopy-мост и обновляются в UI каждые 3 секунды, пока прокси работает.

Надёжность
Прокси не умирает
Три механизма защищают сервис от гибели. START_STICKY просит Android перезапустить сервис после принудительного kill. AlarmManager поднимает его через 3 секунды если пользователь смахнул приложение из недавних. BootReceiver ловит четыре системных события загрузки и стартует прокси сразу после перезагрузки.

Вопросы и ответы
FAQ

Частые вопросы о приложении.

Что такое MTProxy и зачем он нужен? +
MTProxy — официальный протокол прокси для Telegram, разработанный командой Telegram. Он позволяет обойти блокировки Telegram в странах, где мессенджер ограничен. TG WS Proxy поднимает локальный MTProxy-сервер прямо на телефоне и подключает к нему Telegram.
Нужно ли держать приложение открытым? +
Нет. После нажатия «Запустить» сервис работает как Foreground Service в фоне — приложение можно полностью закрыть. Сервис продолжит работать до явной остановки кнопкой «Стоп» или перезагрузки телефона (после перезагрузки он стартует сам через BootReceiver).
Android убивает фоновые процессы — это проблема? +
Именно для этого в v1.0.2 добавлены три уровня защиты: START_STICKY просит Android перезапустить сервис, AlarmManager будит его через 3 секунды после смахивания из недавних, а BootReceiver поднимает прокси после перезагрузки. Вместе они делают сервис практически неубиваемым.
Почему APK такой большой? +
Внутрь APK встроен полноценный Python 3.11 runtime через Chaquopy — отдельно для ARM64 и x86_64. Именно благодаря этому не нужно устанавливать ничего дополнительного: никакого Termux, никаких скриптов. Всё самодостаточно.
Это безопасно? Где исходный код? +
Приложение полностью открыто: исходный код на GitHub, лицензия MIT. Ядро прокси — оригинальный Flowseal/tg-ws-proxy без изменений. Можно проверить каждую строчку кода самостоятельно.
На каких устройствах работает? +
Поддерживаются устройства с Android на архитектурах ARM64 (большинство современных телефонов) и x86_64 (эмуляторы, некоторые нетипичные устройства). Минимальная версия Android зависит от requirementsChaquopy — обычно это Android 5.0+.
